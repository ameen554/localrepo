Git & GitHub Cheat Sheet

Sections grouped by learning stage: Beginner → Intermediate → Advanced

Git - Beginner | Git - Intermediate | Git - Advanced | GitHub - Beginner | GitHub - Intermediate | GitHub - Advanced

Git — Beginner

  • git init — create a new repository
  • git clone <repo-url> — clone a remote repo
  • git status — show working tree status
  • git add <file> — stage changes
  • git add . — stage all changes
  • git commit -m "msg" — commit staged changes
  • git log — show commit history
  • git diff — show unstaged changes

Git — Intermediate

  • git branch — list branches
  • git branch <name> — create branch
  • git checkout <branch> — switch branch
  • git checkout -b <branch> — create & switch
  • git merge <branch> — merge branch into current
  • git rebase <base> — reapply commits on new base
  • git stash — stash changes
  • git stash pop — reapply stashed changes
  • git remote -v — show remotes
  • git fetch — fetch remote refs
  • git pull — fetch + merge (or use --rebase)
  • git push — push commits to remote

Git — Advanced

  • git reset --soft <commit> — move HEAD, keep index
  • git reset --mixed <commit> — default, reset index
  • git reset --hard <commit> — reset working tree (destructive)
  • git revert <commit> — create a new commit that undoes
  • git cherry-pick <commit> — apply a commit onto current branch
  • git reflog — recover lost commits / view HEAD history
  • git bisect — binary search for a bad commit
  • git blame <file> — show who changed each line
  • git submodule add <url> <path> — add a submodule

GitHub — Beginner

  • Create a repo on GitHub web UI
  • Connect local repo: git remote add origin <repo-url>
  • Push initial branch: git push -u origin main
  • Open issues to track bugs/features
  • Use Pull Requests (PRs) to propose changes

GitHub — Intermediate

  • Fork a repo to contribute to someone else's project
  • Use git fetch upstream and git merge to sync forks
  • Protect branches via repository settings (required checks, reviews)
  • Use PR templates and issue templates
  • Use labels, milestones, and projects to organize work
  • Use Actions marketplace for CI workflows

GitHub — Advanced

  • Write GitHub Actions workflows (.github/workflows/*.yml)
  • Use secrets to store tokens (GITHUB_TOKEN or custom)
  • Configure CODEOWNERS to require specific reviewers
  • Use monorepo strategies and submodules/subtrees
  • Use the GitHub API or gh CLI for automation (gh)
  • Manage large files with Git LFS
